# 章节模板（统一写法）

> 适用范围：`算子优化 / 推理引擎 / 大模型架构 / 通信技术` 四大主线章节。

## 1. 一页结论

- 用 3～6 条 bullet 直接回答：这章最重要的结论是什么。
- 要做到 30 秒内可以快速复习。

## 2. 通用知识

建议按下面顺序回答：

1. 它是什么
2. 它解决什么问题
3. 为什么在 AI 推理/系统中重要
4. 它引入了什么代价或 trade-off
5. 它与上下游模块如何关联

## 3. 最小例子

至少给一个可以手算、可以口述、可以在面试中快速展开的例子。

可选形式：

- shape 变化例子
- 显存估算例子
- FLOPs / Bytes 粗估例子
- TTFT / TPOT 拆解例子
- collective 通信例子

## 4. 工程例子

至少给一个系统级例子，最好贴近线上推理或真实 profiling：

- 为什么 TTFT 高
- 为什么 TPOT 高
- 为什么 p99 抖动
- 为什么多卡扩展性差
- 为什么 KV cache / allocator 成为瓶颈

## 5. 常见面试问题

每篇建议 5～8 题，按三层写：

### 初级

- 解释定义
- 讲清楚作用

### 中级

- 比较方案优劣
- 解释 trade-off

### 高级

- 结合场景定位问题
- 解释为什么某个优化在另一个 workload 下不成立

## 6. 易错点

建议聚焦：

- 容易混淆的概念
- 容易只记定义、不懂边界的地方
- 训练视角与推理视角的混淆
- 平均值与尾延迟的混淆

## 7. 排查 checklist

每篇都尽量给出可以执行的 checklist，而不是泛泛而谈。

## 8. 延伸阅读

控制在 3～5 个来源，优先：

- 官方文档
- 经典论文
- 高质量工程实践总结
