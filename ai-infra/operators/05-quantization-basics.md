# 量化基础（INT8/INT4）与误差

## 要点

- 量化的工程目标通常是：更低显存、更高吞吐、更低成本（但需要验证精度）
- 关键决策：量化粒度（per-tensor / per-channel / group-wise）、校准方式、激活是否量化

## 量化的最小概念

- scale/zero-point（仿射映射）
- 对称 vs 非对称量化
- 权重量化 vs 激活量化

## LLM 推理里的常见点

- 权重量化（W8A16、W4A16、W4A8 等组合）
- KV cache 的精度与存储（可能影响吞吐与质量）

## 误差与验证

- 先选一个可复现评测（perplexity/任务集/回归样例）
- 再看线上指标（答案质量、拒答率、毒性过滤等——取决于业务）

## 易错点

- 同一套量化参数对不同分布/不同 batch 可能不稳
- 只看吞吐不看尾延迟（p95/p99）

## 排查 checklist

- [ ] 量化配置写清楚：W?A?、粒度、group size、校准集
- [ ] 性能数据按 shape/批大小分桶统计
- [ ] 精度回归是否可复现（固定随机种子/固定采样策略）
