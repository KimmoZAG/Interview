# AI Infra 学习笔记（Markdown 系列）

目标：用“工程师视角”把 AI 推理/部署相关的核心知识串起来，方便复习与面试。

组织方式：

- **operators/**：基本算子与 kernel（layout、并行、带宽、融合、量化）
- **models/**：模型结构与推理相关机制（Transformer、KV cache、采样等）
- **inference/**：推理框架与工程落地（runtime、图编译、serving、观测与优化）
- **appendix/**：术语、坑、参考

写作约定（建议）：

1. 每篇开头先写 **结论/要点（3-6 条）**。
2. 用“**输入/输出/复杂度/内存访问**”描述算子；用“**张量形状**”贯穿模型与推理。
3. 记录 **瓶颈定位方法**（算力 vs 带宽 vs 同步/launch）与可复现的观测指标。
4. 每篇末尾固定三块：**易错点**、**排查 checklist**、**参考资料**。
