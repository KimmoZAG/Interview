# 预训练数据工程：来源、过滤、去重、质量评估

## 要点

- 从 CS336 Lecture 13-14 的视角看，数据不是“训练前顺手处理一下”的原料，而是决定模型能力边界的核心系统组件。
- 一个可用的预训练数据流水线至少包括：**数据来源 → 解析/抽取 → 过滤 → 去重 → 切分/混合 → 质量评估**。
- 数据工程的目标不是“尽可能多抓数据”，而是让有限的训练预算消耗在更高质量、更相关、更少重复的 token 上。
- 很多公开模型会详细讲架构，却对数据配方保留较多细节；这恰恰说明数据工程在工业界的竞争价值很高。

## 为什么数据工程对 AI Infra 很重要

很多人把数据处理当成建模之外的前处理，但从系统角度看，它直接决定：

- 最终 token 数规模
- 数据管线吞吐与训练输入效率
- 模型是否学到大量垃圾模式
- 后续评测和对齐的上限

也就是说，数据工程同时影响：

- 训练质量
- 训练成本
- 数据加载与存储压力
- scaling law 是否还能成立

## 1. 数据来源

常见来源包括：

- Common Crawl / 网页抓取
- 开源代码仓库与代码转储
- 书籍、论坛、问答、百科、技术文档
- 指令数据、对话数据、合成数据

来源层面最重要的问题不是“量大不大”，而是：

- 分布是否符合目标任务
- 是否包含大量模板化噪声
- 是否存在版权、隐私、合规风险

## 2. 解析与抽取

原始网页或文档通常不能直接进训练。至少要处理：

- HTML / Markdown / PDF / 代码文件解析
- boilerplate 去除
- 主体文本抽取
- 语种识别
- 编码和特殊字符清洗

这一步的目标是：

- 把“原始抓取对象”转成结构稳定的文本片段

如果这一步做得差，后面过滤和去重都会建立在脏输入上，效果会大打折扣。

## 3. 过滤

过滤的目标是提高 token 质量密度，而不是机械删除一切“看起来奇怪”的样本。

常见过滤维度：

- 语言过滤：保留目标语种
- 长度过滤：过短、过长、信息量低的文本剔除
- 重复模板过滤：导航栏、页脚、免责声明、版权说明
- 低质量内容过滤：乱码、spam、自动生成垃圾文本
- 安全/合规过滤：PII、敏感内容、有害内容

一个工程原则：

- 过滤规则必须可重跑、可解释、可抽样检查

否则最后只会得到“数据变少了”，但无法解释为什么变好了或变坏了。

## 4. 去重

CS336 后半段非常强调去重，因为重复数据会同时污染：

- 训练预算利用率
- 泛化能力
- benchmark 泄漏风险

去重通常分两类：

### 精确去重

- 对完全相同的文档或片段做哈希
- 成本较低，解释性强

### 模糊去重

- 处理轻微改写、模板变化、局部重叠
- 常见思路包括 n-gram / MinHash / LSH 等
- 实现更复杂，但对大规模网页语料更关键

一个足够实用的判断：

- 如果不做模糊去重，网页语料里的模板重复和轻微改写内容会严重浪费 token 预算

## 5. 混合与配比

即使每个子数据集都“看起来不错”，最终训练效果也强依赖混合比例。

常见维度：

- 网页文本 vs 代码 vs 数学 vs 对话
- 高质量人工数据 vs 大规模弱监督数据
- 通用语料 vs 领域语料

这一步本质上是在决定：

- 模型把容量优先分配给哪些模式

因此它和架构设计一样，都是能力塑形的一部分。

## 6. 数据质量评估

数据工程最危险的地方在于：

- 你可能做了大量规则，却无法判断它是否真的提高了质量

因此至少要建立几类评估：

### 抽样人工检查

- 规则删掉了什么
- 保留下来了什么

### 统计指标

- 文档长度分布
- 语言分布
- token 长度分布
- 去重比例
- 被过滤比例

### 模型回路验证

- 在小规模训练上比较 validation loss
- 比较下游 benchmark 或 sanity task

很多时候，数据工程是否有效，不在于你规则多不多，而在于小实验能否稳定证明它带来了收益。

## 7. 数据工程与训练预算的关系

从 scaling 和预算的角度看，数据工程做得好，等于：

- 用同样的 token 预算训练更少噪声
- 让有效 token 占比更高
- 减少重复 token 带来的浪费

所以数据工程不是训练前的独立任务，而是预算优化的一部分。

## 8. 一个最小数据流水线

如果你要在笔记里保留一个最小可复述版本，可以记成：

1. 抓取或收集原始数据
2. 解析成干净文本片段
3. 做语言和质量过滤
4. 做精确去重与模糊去重
5. 重新切分和混合数据源
6. 在小模型上做 loss / benchmark 验证

这 6 步已经足以概括绝大多数预训练数据工程主线。

## 你至少要会回答的三个问题

### 例 1：为什么去重会影响最终模型质量

因为重复数据会浪费 token 预算，并可能导致模型过度记忆重复模式。

### 例 2：为什么数据过滤不是越严格越好

因为过度过滤可能损失任务相关分布，造成模型覆盖面下降。

### 例 3：为什么数据质量问题会污染 scaling law

因为一旦数据质量变化，loss 曲线和最优配比不再只由模型大小和 token 数决定。

## 易错点

- 把数据量当成唯一目标，忽略质量密度
- 只做精确去重，不做模糊去重
- 规则写了很多，但没有抽样验证和小实验回路
- 不记录过滤比例和数据分布变化，导致后续无法复盘

## 排查 checklist

- [ ] 当前数据源各自来自哪里，分布有什么偏差？
- [ ] 解析、过滤、去重各自删掉了多少样本？
- [ ] 是否做过抽样人工检查？
- [ ] 是否用小规模训练验证过数据处理改动的收益？
