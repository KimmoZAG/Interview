    11｜Scaling Law 案例：MUP、WSD 与工程外推¶

    原始来源：https://tuananhbui89.github.io/blog/2025/cs336-lec11/

    先抓住这讲要点¶

    • 真正可落地的 scaling workflow，不只是“画一条 log-log 直线”，而是 稳定参数化 + 可迁移超参 + 合理训练日程 + 小模型代理实验 + 大模型确认 的完整流程。
    • MUP（Maximal Update Parameterization）解决的是：模型宽度变了以后，为什么小模型上调好的超参常常不再好用。
    • WSD（Warmup-Stable-Decay）解决的是：如何让 scaling 实验更稳定、更可复用、更适合 checkpoint rewinding 和 token budget 分析。

    这一讲在整门课里的位置¶

    第 9 讲告诉我们 scaling law 是一种预测工具。
    但真正做工程的人很快会追问：

    好，理论上我知道该拟合趋势了，可为什么我一换模型宽度，学习率就崩了？为什么同样的 token budget 实验这么难公平比较？

    这就是第 11 讲存在的意义：它把“scaling law 的数学直觉”进一步推进到“实验方法论”。

    换句话说，这一讲解决的是：

    • 如何让小模型实验更能代表大模型；
    • 如何让不同 token budget 的实验比较更公平；
    • 如何减少因为参数化和 schedule 造成的结论漂移。

    这讲想训练你什么能力¶

    学完这一讲，你应该能：

    • 看懂为什么很多 scaling 论文不是只谈曲线，还会谈参数化和学习率；
    • 理解 MUP 这种方法的目标不是“提高上限”，而是“提高可迁移性”；
    • 知道为什么 WSD 这类 schedule 在工程实验中非常受欢迎；
    • 明白 tokens/params 比例不是固定真理，而是和训练配方强耦合的结果。

    代表图¶

    为什么“只做小模型实验”经常会翻车¶

    很多团队一开始都抱着一个很自然的期待：

    1. 先在小模型上调好超参；
    2. 再把模型放大；
    3. 直接复用那套配置。

    听起来很合理，但现实常常是：

    • 小模型最优学习率，到大模型上不再最优；
    • 宽度一变，梯度统计性质也变；
    • 初始化、层更新尺度、激活分布都可能变化；
    • 结果是“小模型上看起来完美”的设置，到了大模型就不稳。

    所以真正的问题不是“能不能缩小做实验”，而是：

    我怎样设计小模型，使它在优化行为上更像将来的大模型？

    这正是 MUP 这类方法要解决的核心。

    MUP 在解决什么问题¶

    普通参数化的麻烦¶

    在普通参数化下，当你改变模型宽度时，不同层参数更新的相对尺度可能跟着变。
    这会导致：

    • 某些层更新过大；
    • 某些层更新过小；
    • 原来稳定的学习率，在更宽模型上突然不稳定或效果变差。

    于是你就很难做到：

    在小模型上调一次参，然后放心迁移到更大模型。

    MUP 的直觉¶

    MUP 的目标不是让模型“神奇变强”，而是让不同宽度下的优化动力学更可比。
    它通过合适的参数化、初始化和学习率缩放规则，尽量让：

    • 小模型和大模型的更新行为保持某种一致性；
    • 超参数的最优区域不要随宽度大幅漂移；
    • scaling 实验更有可迁移性。

    一句人话版总结：

    MUP 不是在追求一条更高的曲线，而是在追求“曲线之间更能对得上”。

    一个直观的 MUP 心智模型¶

    你可以把训练看作“每层都在用自己的步幅前进”。
    如果模型变宽后，某些层突然迈大步，某些层突然迈小步，那原来调好的全局学习率自然就不再合适。

    MUP 想做的是：

    • 当宽度变化时；
    • 不同层的更新尺度关系尽量保持稳定；
    • 于是原本在小模型上工作的学习率和 schedule，更可能继续有效。

    它本质上是在提升“小到大”的实验一致性。

    WSD 为什么适合 scaling 实验¶

    如果说 MUP 关心的是“不同宽度之间怎么更可比”，那 WSD 关心的是：

    一条训练曲线本身，怎样更适合做实验分析和预算比较？

    WSD 通常把训练分成三段：

    1. Warmup：逐步升到目标学习率；
    2. Stable：保持一个稳定 plateau；
    3. Decay：在后段快速下降收敛。

    它为什么工程上好用¶

    因为它把训练过程切成了几个很清晰的阶段：

    • warmup 阶段解决初期不稳定；
    • stable 阶段提供一个更平稳的观察窗口；
    • decay 阶段负责把模型收束到更好终点。

    这比某些“全程平滑但阶段感不强”的 schedule 更适合做实验分析，尤其在下面这些场景里：

    • token budget 对比；
    • checkpoint rewind；
    • 中途截断实验；
    • 不同规模模型的公平比较。

    代码拆解：WSD schedule 示意¶

    def wsd(step, warmup, stable_end, total, lr):
    if step < warmup:
        return lr * step / warmup
    if step < stable_end:
        return lr
    ratio = (step - stable_end) / (total - stable_end)
    return lr * (1 - ratio)

    这段代码看上去比 cosine 还“朴素”，但它很有工程味：

    • warmup 段避免一上来把模型打懵；
    • stable 段让你有足够长的稳定训练区间做比较；
    • decay 段负责最后的收敛收口。

    所以 WSD 的关键不是数学形式多精美，而是：

    它给了你一个更适合做实验切片和回放分析的训练轨迹。

    为什么 WSD 对 checkpoint rewinding 特别友好¶

    这是非常实用的一点。
    如果你已经跑了一条较长训练曲线，并且中间保留了多个 checkpoint，那么在 WSD 的 stable 区间里：

    • 模型处于相对可比较的训练状态；
    • 可以从不同 checkpoint 出发继续训练；
    • 近似模拟不同 token budget 下的表现。

    这样你就不必为每个 token budget 都从头重新训练一遍。
    对于昂贵实验来说，这种复用能力非常值钱。

    为什么不同论文的 tokens / params 比例差别很大¶

    这是很多人学习 scaling law 时最容易误解的地方。
    看到某篇论文说最优比例大概是 20:1，就很容易把它当成一条固定法则。
    其实它高度依赖训练配方。

    影响比例的因素至少包括：

    • 架构（例如 attention 设计、FFN 比例、norm 选择）；
    • 数据质量与去重程度；
    • tokenizer；
    • 学习率 schedule；
    • 优化器与正则；
    • 拟合方法和外推目标；
    • 用什么指标评估“最优”。

    所以更好的理解方式是：

    token/param 比例不是圣经，而是某一类模型、某一套数据、某一种训练方法下得到的经验解。

    一个更完整的 scaling workflow 长什么样¶

    把这一讲和第 9 讲连起来，一个成熟的 workflow 往往像这样：

    1. 选一套相对稳定的参数化方式；
    2. 选一条适合分析的 LR schedule；
    3. 在小模型上做代理实验；
    4. 检查超参是否具有可迁移性；
    5. 拟合趋势并做外推；
    6. 用少量大模型实验验证关键结论；
    7. 再把结论转成预算和配方决策。

    这和“画一条线就相信世界”的差别非常大。
    前者是工程方法论，后者更像美图秀秀科研版。

    面试里可以怎么讲¶

    如果面试官问：“MUP 的核心价值是什么？”

    你可以答：

    MUP 的核心价值是提高超参数在不同模型宽度之间的可迁移性。普通参数化下，模型变宽后最优学习率和更新尺度常常漂移，导致小模型上调好的配置不能直接迁移。MUP 通过更稳定的参数化和缩放规则，让不同宽度模型的优化行为更可比。

    如果面试官问：“为什么 WSD 对 scaling 实验很有用？”

    可以答：

    因为它把训练分成 warmup、stable、decay 三段，stable 区间特别适合做 token budget 对比和 checkpoint rewinding，使实验更容易复用、更可比，也更利于外推分析。

    复习题¶

    1. 为什么“小模型调参后直接放大”常常会失效？
    2. MUP 想稳定的到底是什么：模型上限，还是超参可迁移性？
    3. WSD 相比很多传统 schedule，更适合哪类实验分析？
    4. 为什么 checkpoint rewinding 能显著节省 scaling 实验成本？
    5. 为什么 20:1 这类 token/param 比例不能被当成普适真理？
