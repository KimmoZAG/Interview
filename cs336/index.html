    CS336 中文笔记¶

    这是基于 Stanford CS336 公开课程整理的一套中文讲义。它的目标不是逐句翻译，而是把课程内容改造成更适合中文读者的系统化学习材料、工程复习资料和面试准备底稿。

    这套材料的核心定位有三点：

    • 讲清主线：把模型、系统、数据、对齐放在一条完整链路里讲。
    • 保留工程感：尽量把结论落到 FLOPs、显存、通信、吞吐、延迟这些真实约束上。
    • 兼顾面试准备：每讲尽量提供可复述的解释框架，而不是只留抽象结论。

    从哪里开始¶

    • 课程总索引： 00-index.md
    • 学习路线图： study-roadmap.md
    • 面试准备指南： interview-prep-guide.md

    这套笔记适合谁¶

    如果你属于下面任意一类读者，这套材料会比较合适：

    • 想系统补大模型基础，而不是停留在 API 调用层。
    • 想把训练、推理、并行、数据、对齐放在同一张脑图里。
    • 准备大模型基础设施、训练系统、推理优化、算法工程类面试。
    • 已经看过零散论文，但缺少一套能串起来的中文资料。

    建议怎么用¶

    第一次系统学习¶

    建议按 00-index.md 的顺序，从 01 到 17 读完一遍，先建立整体结构，再回头补细节。

    面试准备¶

    建议优先看下面这些讲次：

    • 01：课程总览与 tokenizer
    • 02：FLOPs、显存、资源核算
    • 03：Transformer 默认结构与超参
    • 05：GPU、roofline、memory bound
    • 07-08：并行训练、通信与系统拆分
    • 09-11：scaling law 与资源分配
    • 10：推理优化与 KV cache
    • 13-14：数据工程与过滤去重
    • 15-17：SFT、RLHF、DPO、GRPO

    建议别只按“记知识点”去复习，而要优先练下面三类表达：

    • 公式能不能推：例如 6PN、KV cache 量级、显存字节账
    • 机制能不能讲：例如 view / contiguous、prefill / decode、FSDP gather / reshard
    • 决策能不能落地：例如为什么某个模型必须上分布式、为什么某个优化更适合长上下文场景

    工程实现¶

    建议沿着下面这条线读：

    02 -> 05 -> 06 -> 07 -> 08 -> 10 -> 14 -> 16 -> 17

    如果你是带学员或做团队知识沉淀，建议把每一页都当成一个 few-shot 样本来用：

    • 找到一段“只有结论”的笔记
    • 补推导、补数字、补决策
    • 再让学员照这个颗粒度重写自己的版本

    这个方法通常比泛泛讲“要更工程化”有效得多。

    研究视角¶

    建议沿着下面这条线读：

    01 -> 03 -> 04 -> 09 -> 11 -> 12 -> 13 -> 15 -> 16 -> 17

    这次重构后的组织方式¶

    为了让 cs336 更像一套完整课程资料，而不是单篇笔记堆叠，现在分成四层：

    1. README：解释这套材料的定位、使用方式和入口。
    2. 00-index：课程总地图，说明每讲在整条链路中的作用。
    3. study-roadmap：按目标导向组织阅读路径，适合系统学习和阶段复习。
    4. interview-prep-guide：把课程知识重新整理成面试会真的问的主题簇。

    这套材料强调什么¶

    CS336 的价值不只是“知道术语”，而是学会一种统一分析框架：

    任何一个设计，都要同时回答三个问题：它提高了什么能力，它消耗了什么资源，它引入了什么系统代价。

    所以阅读这套笔记时，建议你不断追问：

    • 这个结论在哪个规模区间成立？
    • 它改善的是 accuracy、throughput、latency 还是 cost？
    • 它的代价是在算力、显存、通信、数据质量，还是工程复杂度上？
