    15｜对齐（一）：SFT 与 RLHF 全景¶

    原始来源：https://tuananhbui89.github.io/blog/2025/cs336-lec15/

    先抓住这讲要点¶

    • 预训练模型不等于可用助手。真正变成 ChatGPT 风格产品，通常必须经过 post-training。
    • SFT（Supervised Fine-Tuning）负责把“会续写文本”变成“会按指令输出符合格式的回答”；RLHF 进一步把行为往“人类更偏好”的方向推。
    • 高质量对齐数据并不等于“越华丽越好”，如果示范答案超出了 base model 的真实能力边界，反而可能教出更会模仿格式的幻觉模型。

    这一讲在整门课里的位置¶

    前面几讲讲的是预训练、数据、评测和系统。
    但大家真正熟悉的大模型产品体验——会听指令、会拒答、会按格式写、会更像助手——很多并不是预训练天然给的。

    这一讲回答的是：

    为什么一个“会语言”的 base model，还需要额外的后训练，才能变成用户真正觉得好用的助手？

    这讲想训练你什么能力¶

    学完这一讲，你应该能：

    • 分清 pre-training 和 post-training 在目标上的根本差别；
    • 理解 SFT 和 RLHF 在产品模型中的分工；
    • 看懂 InstructGPT 这条经典流程为什么会成为范式；
    • 知道为什么 SFT 数据质量不是“越会写越好”，而是要考虑真实性和能力支撑。

    代表图¶

    为什么预训练模型不等于可用助手¶

    一个纯 pre-training 的语言模型，通常很擅长做一件事：

    • 延续文本分布；
    • 预测下一个 token；
    • 在很多知识和语言模式上表现出惊人的通用性。

    但这不等于它天然擅长：

    • 理解“这是用户指令”；
    • 稳定遵循格式要求；
    • 进行安全拒答；
    • 在多轮对话里保持合适风格；
    • 输出用户真正偏好的答案形态。

    所以预训练模型更像“会说很多语言规律的人”，而不是“已经学会怎么做一个产品助手的人”。

    Post-training 到底在补什么¶

    后训练的核心不是重新教知识，而是重新塑造行为接口。
    它主要在补下面几类能力：

    • 指令遵循；
    • 回答格式；
    • 有用性；
    • 安全性；
    • 人类偏好对齐；
    • 多轮交互风格。

    从产品角度看，它回答的是：

    模型不仅要“会”，还要“会按我们希望的方式表现出来”。

    InstructGPT 三步走为什么经典¶

    后训练时代最著名的一条范式路线就是 InstructGPT 风格三步：

    1. SFT：用高质量示范把模型拉到“会按指令答题”的区域；
    2. Reward Model：从人类偏好中学习“什么答案更好”；
    3. RL：让策略朝更高偏好得分方向优化。

    这套流程为什么强？
    因为它把“让模型更像用户想要的助手”拆成了三个更可操作的问题：

    • 先学会基本行为格式；
    • 再学会偏好排序；
    • 最后做策略层面的强化。

    SFT 在本质上做了什么¶

    SFT 并不神秘。
    本质上它仍然是监督学习，只不过训练样本从普通文本续写变成了：

    • prompt → response；
    • instruction → demonstration；
    • 多轮对话 → target assistant turn。

    代码拆解：SFT 的最小损失¶

    def sft_loss(logits, labels):
    # 常规 next-token cross entropy
    return cross_entropy(logits[:, :-1], labels[:, 1:])

    这段代码特别重要，因为它告诉你：

    SFT 不是换了一个神秘目标函数，而是换了一种更有行为约束的数据格式。

    也就是说，post-training 的第一步，其实是用数据把模型输出接口重新塑形。

    为什么 instruction data 不是越多越好¶

    这是很多人对 SFT 的第一个误区。
    直觉上好像“多喂一些看起来厉害的答案”就会更好，但现实不是这样。

    因为小而精的数据对行为有很强杠杆¶

    SFT 数据量往往远小于预训练数据，但它对模型行为风格的影响却非常大。
    所以如果这部分数据出了问题，模型会很快被带偏。

    什么叫“看起来高质量，实际上危险”¶

    有些答案非常像人类专家写的：

    • 文风漂亮；
    • 结构完整；
    • 引用丰富；
    • 语气自信。

    但如果 base model 本身并没有足够知识和推理能力支撑这些表达，那么你教给模型的可能不是“更真实”，而是“更会装”。

    结果就是：

    • 更会模仿 citation；
    • 更会写得像真的；
    • 但不一定更可靠；
    • 甚至更容易产生高置信幻觉。

    所以高质量 SFT 数据真正要追求的，不是“答案很华丽”，而是：

    • 真实；
    • 一致；
    • 对 base model 能力边界友好；
    • 符合目标产品行为。

    为什么 RLHF 这么有吸引力¶

    如果什么都靠人类写标准答案，会非常贵：

    • 成本高；
    • 覆盖难；
    • 很多开放式任务根本没有单一标准答案。

    但让人类做“二选一偏好判断”，通常更容易：

    • 哪个答案更有帮助；
    • 哪个更安全；
    • 哪个更清楚；
    • 哪个更符合任务目标。

    这比要求人类每次都写出完美答案更便宜，也更稳定。

    所以 RLHF 的核心魅力是：

    它把难以直接写成标签的“偏好”，变成了更容易收集的比较信号。

    代码拆解：pairwise preference 的数据形式¶

    sample = {
    "prompt": "Explain attention",
    "chosen": "Good answer...",
    "rejected": "Bad answer...",
}

    这是后面对 DPO、RM、RLHF 都非常重要的数据基元。
    它表达的是：

    • 同一个 prompt；
    • 至少两个候选回答；
    • 人类或某种评判机制认为其中一个更好。

    这类 pairwise preference 非常关键，因为它比“绝对分数”更稳定，也更接近真实选择行为。

    一个更完整的心智模型：从“会说”到“会按你想要的方式说”¶

    可以把整个 post-training 理解成三层递进：

    1. SFT：教模型学会任务格式；
    2. 偏好建模：教模型知道什么更讨喜、更有帮助；
    3. RL / preference optimization：把行为分布整体往偏好更高的区域推。

    所以后训练不是在“凭空造能力”，而是在：

    • 调整输出接口；
    • 放大已有能力；
    • 抑制不想要的行为模式；
    • 让模型更适合真实用户场景。

    面试里可以怎么讲¶

    如果面试官问：“为什么 post-training 对产品模型几乎是必须的？”

    你可以答：

    因为预训练模型主要学到的是语言建模分布，不会天然以产品助手的方式响应用户。post-training 通过 SFT 和偏好优化，把模型从“会续写文本”变成“会遵循指令、符合格式、具备更好交互风格和安全行为”的系统。

    如果面试官问：“为什么高质量 SFT 数据不一定越华丽越好？”

    可以答：

    因为如果示范答案的表现超出了 base model 的真实支撑能力，模型学到的可能是表面格式和自信表达，而不是真实可靠性，这反而会加剧高置信幻觉。

    复习题¶

    1. 为什么预训练模型往往还不足以直接成为可用助手？
    2. SFT 和 RLHF 在 post-training 里的角色分别是什么？
    3. 为什么说后训练更像“行为塑形”，而不只是“继续喂知识”？
    4. 为什么 pairwise preference 在工程上常比 full demonstration 更经济？
    5. 为什么过于“华丽”的 SFT 数据有时会带来副作用？
