    AI Infra¶

    工程师视角的 AI 基础设施知识体系，以"能做出正确系统判断"为核心目标。

    • 算子优化

      单算子 / 子图 / Kernel 为什么快、为什么慢、如何定位和优化。

      进入

    • 推理引擎

      模型如何从请求进入，经过 runtime / compiler / scheduler / KV cache，最终稳定产生 token。

      进入

    • LLM 架构

      推理优化工程师需要懂到什么程度的模型知识，才能做出正确的系统判断。

      进入

    • 分布式通信

      多卡系统为什么慢、慢在什么通信模式、为何通信会比计算更早成为瓶颈。

      进入

    • 附录

      术语表、坑位清单，以及 Scaling Laws、数据工程、Post-training 等扩展专题。

      进入

    各节概览¶

    算子优化¶

    篇目 核心问题
    张量与内存布局 stride、contiguous、view 与 copy 的代价
    Kernel 执行模型 SIMD / SIMT / warp / block 和 occupancy
    内存层级与 Roofline arithmetic intensity，memory-bound vs compute-bound
    计算图融合与调度 op fusion、kernel launch overhead、调度策略
    量化基础 INT8/INT4、calibration、量化误差
    FlashAttention 与 IO-aware tiling、重计算、IO 感知的 Attention 实现

    推理引擎¶

    篇目 核心问题
    推理栈全景 前端 → IR → 编译 → Runtime 全流程
    ONNX Runtime / TensorRT session、execution provider、TRT plan
    图编译 TVM/MLIR/XLA IR 设计、pass 流水线、代码生成
    LLM Serving continuous batching、KV cache 管理、常见框架
    推理优化 Playbook 定位 → 动作 → 验证的系统方法
    可观测性与调试 TTFT / TPOT / p95、tracing、profiling
    Paged KV 与 Allocator PagedAttention、块管理、显存稳定性
    长上下文推理 chunked prefill、context 切分、扩展策略

    LLM 架构¶

    篇目 核心问题
    Transformer 最小知识 推理视角下必须懂的 Transformer
    Attention 与 KV Cache MHA/GQA/MLA，KV Cache 的成本来源
    Norm/激活/数值稳定性 LayerNorm/RMSNorm、SwiGLU、梯度问题
    Tokenizer 与采样 BPE、temperature、top-p、speculative decoding
    评测与基准 perplexity、MMLU、GPQA、latency / throughput
    MoE 最小导读 稀疏激活、路由、系统代价
    训练资源核算 FLOPs、显存、tokens 的账怎么算

    分布式通信¶

    篇目 核心问题
    并行训练策略 DP / TP / PP / SP / FSDP / ZeRO
    通信基础 latency / bandwidth / topology
    互联与拓扑 PCIe / NVLink / NVSwitch / InfiniBand / RDMA
    集合通信原语 all-reduce / all-gather / reduce-scatter / all-to-all
    并行到通信映射 每种并行策略引入的通信体积与频次
