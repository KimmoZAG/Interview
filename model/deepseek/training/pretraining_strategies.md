# Pretraining Strategies

## 关键结论

DeepSeek 的预训练并不是“把更多 token 喂给更大的模型”这么简单，而是在回答一个更现实的问题：**训练预算有限时，怎样让每个 token 更值钱。**

- 它先从数据质量下手，而不是先盲目扩语料规模 [DeepSeek LLM, Section 2.1; DeepSeek-V2, Section 3.1.1]。
- 它把长上下文扩展做成分阶段调度，而不是一上来全部拉满 [DeepSeek-V2, Section 3.1.4; DeepSeek-V3, Section 4.3]。
- 到 V3，它又用 `MTP` 增加训练信号密度，让模型不只学“下一个 token”，还更早学会对未来做准备 [DeepSeek-V3, Section 2.2]。

所以这一页最重要的结论是：**DeepSeek 的预训练优势，不只是“喂得多”，而是“更少浪费、更早为 reasoning 铺路”。**

## 背景：为什么预训练不能只看 token 数量

### 旧做法为什么不够

当模型进入万亿 token 级别后，预训练的主要矛盾就不再只是“有没有更多数据”，而是：

- 重复数据会不会持续浪费训练预算；
- 低质量、低信息密度语料会不会让梯度变得便宜但没用；
- 长上下文和更强训练目标会不会把训练稳定性先压垮；
- 模型会不会只学会局部 next-token，而不是更强的长程规划能力。

也就是说，旧范式的问题不在于它错，而在于它太容易把预算花在“看起来很多、其实不太值钱”的 token 上。

### DeepSeek 这一页想解决什么

这一页真正要回答四件事：

1. DeepSeek 为什么这么重视去重、过滤和重混；
2. 为什么长上下文扩展要分阶段做；
3. 为什么 `MTP` 会成为 V3 的关键训练目标升级；
4. 这些动作到底怎样为后续 reasoning 能力打底。

## DeepSeek 具体怎么做

### 第一步：先提升 token 的有效信息密度

DeepSeek LLM 很早就把数据工程拆成三步 [DeepSeek LLM, Section 2.1]：

- `Deduplication`：减少重复文档和重复梯度；
- `Filtering`：过滤低质量、低信息密度内容；
- `Remixing`：重新平衡不同来源和领域的数据比例。

到 V2 和 V3，这个方向进一步强化：

- 提高高质量多源数据占比；
- 增加中文、数学、代码与多语种覆盖；
- 尽量减少冗余，同时保留语料多样性 [DeepSeek-V2, Section 3.1.1; DeepSeek-V3, Section 4.1]。

这背后的逻辑很简单：**不是所有 token 都同样有用，预训练首先要做的是减少低价值 token 对预算的侵占。**

### 第二步：把长上下文扩展做成阶段化调度

DeepSeek 没有把“长上下文”当成一开始就要拉满的默认目标。

- V2 是先完成主预训练，再扩到 `128K` [DeepSeek-V2, Section 3.1.4]；
- V3 更明确，分成 `32K -> 128K` 两阶段扩展 [DeepSeek-V3, Section 4.3]。

这么做的原因很工程化：

- 长上下文会显著抬高显存和通信成本；
- 如果模型基础分布还没学稳，就会让训练信号更稀、更不稳；
- 先学会“正常看”，再学会“看更长”，性价比更高。

### 第三步：用 MTP 提高训练信号密度

V3 最值得注意的升级，是引入 `Multi-Token Prediction (MTP)` [DeepSeek-V3, Section 2.2]。

标准自回归预训练只优化：

$$
\mathcal{L}_{\mathrm{NTP}} = -\frac{1}{T} \sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t})
$$

而 MTP 的直觉是：**同一个位置，不只监督下一个 token，还要给更远一步的未来提供训练信号。**

DeepSeek 的做法不是粗暴多加几个 head，而是用顺序式模块，在保持因果链的前提下往未来多看一步 [DeepSeek-V3, Section 2.2]。

这类设计的意义主要有两点：

1. 训练信号更密；
2. 模型表示更容易学到“对未来做准备”的倾向。

### 这套设计带来的直接优点

把这套预训练策略压缩成收益清单，大概是三类：

- **更少预算浪费**：重复、低质量 token 占比下降；
- **更稳的长上下文能力增长**：上下文扩展不再一口气把系统压垮；
- **更强的 reasoning 底座**：数学、代码和未来规划式表示在 base model 阶段就被更积极地准备好了。

## 数据怎么说明这些优点

### 证据一：V2 和 V3 都在持续提高高价值数据占比

DeepSeek-V2 明确提到继续提高高质量多源数据占比、增加中文数据、改进质量过滤、减轻区域文化偏置 [DeepSeek-V2, Section 3.1.1]。V3 则进一步强调：

- 增加数学与编程样本比例；
- 扩展多语种覆盖；
- 尽量减少冗余，同时保持多样性 [DeepSeek-V3, Section 4.1]。

这些动作本身就是证据：**DeepSeek 认为预训练收益的关键，不在 token 总数，而在 token 结构。**

### 证据二：长上下文被明确做成阶段化扩展，而不是默认配置

V2 的 `128K` 扩展和 V3 的 `32K -> 128K` 两阶段方案都说明了同一件事：DeepSeek 没把长上下文当成“开关一拨就行”，而是当成要分阶段控制风险的训练目标 [DeepSeek-V2, Section 3.1.4; DeepSeek-V3, Section 4.3]。

这说明他们关心的不是“最长能到多少”，而是“怎么在可承受成本里把它稳定练出来”。

### 证据三：MTP 被写进主文，说明它不是附赠技巧

V3 在主文里明确介绍 `MTP`，并且把它与训练信号密度、未来规划式表示和 speculative decoding 价值联系起来 [DeepSeek-V3, Section 2.2]。

这说明 MTP 不是小修小补，而是 DeepSeek 对“预训练目标应该怎样为后续 reasoning 铺路”的一次明确回答。

## 思考问题

- 如果训练预算固定，你会优先投资更高质量数据、长上下文，还是 MTP？为什么？
- DeepSeek 这种阶段化扩展，更像课程学习，还是更像系统风险控制？
- 如果数学/代码数据占比继续提高，哪些能力最可能受益，哪些能力可能被挤压？
