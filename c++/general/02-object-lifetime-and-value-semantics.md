# 对象/生命周期/值语义

## 核心定义（What & Why）

> **一句话总结**：对象生命周期和存储期决定了“这个东西什么时候存在、什么时候销毁、谁负责它的资源”，而值语义决定了“拷贝之后是不是各管各的”，这是写出安全、可推理 C++ 代码的地基。

## 要点

- C++ 里“对象”不仅是类实例，也包括标量、数组、临时对象等
- 重点掌握：**构造/析构时机**、**拷贝/移动**、**引用折叠**（后续与完美转发相关）
- 值语义：像 `int` 一样“拷贝即独立”，是很多良好设计的目标

## 核心代码 / 操作演示（How）

### 返回值与生命周期

```cpp
#include <string>

std::string make_name() {
	 std::string s = "copilot";
	 return s; // 现代 C++ 中这里通常会发生拷贝省略或移动
}
```

### 典型错误：返回局部对象引用

```cpp
const std::string& bad_ref() {
	 std::string s = "dangling";
	 return s; // 错：函数返回后 s 已析构
}
```

关键不是死记“这行错了”，而是建立判断：

- 对象的存储期是什么？
- 引用 / 指针是否活得比对象更久？
- 这里到底该返回值、引用，还是智能指针？

## 对比表

| 概念 | 含义 | 典型收益 | 典型风险 |
|---|---|---|---|
| 自动存储期 | 作用域结束自动销毁 | 简单、高效、天然 RAII | 返回引用/指针容易悬垂 |
| 静态存储期 | 程序结束前一直存在 | 全局共享、初始化一次 | 跨翻译单元初始化/析构顺序坑 |
| 动态存储期 | 手动或智能指针管理 | 生命周期灵活 | 容易泄漏、悬垂、重复释放 |
| 值语义 | 拷贝后各自独立 | 易推理、组合性好 | 大对象拷贝成本可能高 |
| 引用语义 | 多处指向同一对象 | 避免复制、共享状态 | 生命周期和别名问题更复杂 |

## 对象的存储期

- 自动存储期（栈上）：作用域结束析构
- 静态存储期：程序结束析构（顺序有坑）
- 动态存储期（堆上）：`new/delete` 或智能指针管理

## 拷贝/移动与 Rule of X

- Rule of 0：优先让成员自己管理资源（如 `std::vector`），类本身不写特殊成员
- Rule of 5：自管资源时，通常要考虑析构 / 拷贝构造 / 拷贝赋值 / 移动构造 / 移动赋值
- 实战里，优先争取 Rule of 0；Rule of 5 往往意味着你已经在直接碰资源边界

## 💥 实战踩坑记录（Troubleshooting）

> 现象：代码能编译，但运行时偶发崩溃，打印出来的字符串内容像乱码。

- **误判**：以为是编码问题、线程问题，或者某处内存被“神秘覆盖”。
- **根因**：函数返回了局部对象的引用或指针，导致典型悬垂引用。
- **解决动作**：
  - 先追对象的创建位置和析构时机；
  - 确认返回类型到底该是值、引用还是拥有型对象；
  - 对局部构造的结果，优先直接返回值。
- **复盘**：生命周期 bug 最烦的地方就在于——它经常编译没问题、静态看也像没问题，但运行时才露獠牙。

> 常见异常：把对象 `std::move` 之后还继续假定它保留原值。

- moved-from 对象必须仍保持“有效但未指定状态”；别把它当成还没被搬空的原对象来依赖。

## 易错点

- 返回引用/指针指向局部变量
- 误用 `std::move`（把还能用的对象提前“搬空”）
- 静态对象析构顺序（跨翻译单元）

## 🎯 面试高频 Q&A

### 初级

1. 值语义是什么意思？
	- 回答要点：对象拷贝后拥有独立状态，像 `int` 一样“拷贝即独立”；优点是更容易推理和组合。
2. 自动存储期和动态存储期有什么区别？
	- 回答要点：自动存储期随作用域结束自动析构；动态存储期由程序显式管理或交给智能指针管理。

### 中级

1. 为什么现代 C++ 倾向于返回值而不是返回裸指针/引用？
	- 回答要点：返回值更安全，依赖拷贝省略和移动语义通常成本可接受，还能减少生命周期歧义。
2. Rule of 0 和 Rule of 5 分别说明什么设计倾向？
	- 回答要点：Rule of 0 倾向组合标准资源管理类型；Rule of 5 表示类自己直接管理资源，需要完整处理特殊成员函数。

### 高级

1. 什么时候会发生拷贝省略？
	- 回答要点：典型场景是返回局部对象、直接用临时对象初始化目标对象；C++17 起部分情形是强制省略。
2. `const T&` 绑定临时对象时，生命周期延长规则是什么？
	- 回答要点：临时对象可延长到该引用的生命周期末尾，但这不适用于所有转发和成员子对象场景，回答时要强调规则有边界。
