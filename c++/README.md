# C++ 学习笔记（Markdown 系列）

这套笔记按两条主线组织：

- **general/**：与版本无关的通用知识（对象模型、RAII、模板、STL、并发、工具链等）
- **versions/**：按标准版本梳理“新增/变化点”，并配最小可运行示例与易错点

建议写作规则（保持可持续）：

1. 每篇只解决一个主题；开头先写 **结论/要点**，再写细节。
2. 每个知识点至少配一个 **最小例子**（能编译/能运行）。
3. 记录 **易错点/边界条件** 与 **面试追问**。
4. 看到新坑就追加到 `appendix/gotchas.md`，定期回填到对应主题。
